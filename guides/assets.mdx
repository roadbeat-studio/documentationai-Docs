## Overview

The **Asset Library** is Studio's central media management system. It handles images, videos, documents, and other files with automatic metadata extraction, folder organization, tagging, and usage tracking.

<Callout kind="info">
  All assets are **org-scoped**. Uploaded files are stored under `{orgSlug}/{year}/{month}/` paths, and the `/uploads` endpoint enforces org-ownership for authenticated requests. See [Multi-Tenancy](/guides/multi-tenancy#asset-storage-isolation) for details.
</Callout>

## Uploading Assets

<Tabs>
  <Tab title="Admin UI" icon="monitor">
    1. Navigate to **Assets** in the sidebar
    2. Click **Upload** or drag files directly into the browser
    3. Select one or more files (max 50 MB per file)
    4. Optionally choose a target folder
    5. Files are uploaded and metadata is extracted automatically
  </Tab>
  <Tab title="API" icon="code">
    ```bash
    curl -X POST http://localhost:3000/api/v1/assets/upload \
      -H "Authorization: Bearer YOUR_JWT_TOKEN" \
      -F "file=@photo.jpg" \
      -F "folderId=folder-uuid"
    ```
  </Tab>
</Tabs>

### Automatic Metadata Extraction

When you upload an image, Studio automatically extracts:

- **Dimensions** — width and height in pixels
- **EXIF data** — camera model, exposure, GPS coordinates, etc.
- **File info** — MIME type, file size, original filename

## Organizing with Folders

Assets can be organized into a folder hierarchy. The folder sidebar on the left provides full management capabilities:

- **Create folders** — click the `+` button in the sidebar header, enter a name
- **Rename folders** — hover over a folder to reveal the pencil icon; click to edit inline (Enter to save, Escape to cancel)
- **Delete folders** — hover to reveal the trash icon; only empty folders can be deleted (move or delete contents first)
- **Move assets to folders** — drag asset cards from the grid onto a folder in the sidebar, or use the **Move** button in the detail drawer to open a folder picker modal
- **Browse folders** — click a folder in the sidebar to filter the grid to that folder's contents; click "All files" to show everything

```bash
# Create a folder
POST /api/v1/assets/folders
{
  "name": "2026 Campaign",
  "parentId": null
}

# Rename a folder
PATCH /api/v1/assets/folders/:id
{ "name": "Thumbnails" }

# Delete an empty folder
DELETE /api/v1/assets/folders/:id

# Move an asset to a folder
PATCH /api/v1/assets/:id
{ "folderId": "folder-uuid" }

# Move an asset to root (no folder)
PATCH /api/v1/assets/:id
{ "folderId": null }

# List folders
GET /api/v1/assets/folders
```

## Tagging

Tags help you categorize and find assets quickly:

- Add multiple tags to any asset
- Browse popular tags across the organization
- Filter assets by tag in the search

```bash
# Add tags
POST /api/v1/assets/:id/tags
{ "tags": ["hero-image", "campaign-2026"] }

# Remove tags
DELETE /api/v1/assets/:id/tags
{ "tags": ["outdated"] }

# Get popular tags
GET /api/v1/assets/tags?limit=20
```

## Search, Sort & Filter Bar

The Asset Library includes a unified filter bar above the asset grid with the following controls:

| Control | Description |
|---------|-------------|
| **Search** | Debounced (300ms) filename search — case-insensitive contains match |
| **Sort** | Dropdown with Newest first, Name, Size, and Type |
| **Type filter** | Dropdown with All types, Images, Videos, Audio, PDF, Documents |
| **View mode** | Three-mode toggle: Small tiles, Large tiles (default), List view |

### View Modes

- **Small tiles** — compact `auto-fill` grid with a minimum tile width of 120 px; fills the full available width so you get more columns on wider screens. Drag handles, file-type icons, and minimal labels. Page size scales with viewport width.
- **Large tiles** (default) — `auto-fill` grid with a minimum tile width of 180 px; displays drag handles, full filenames, and file sizes. Page size scales with viewport width.
- **List view** — compact rows with a 40×40 px thumbnail, filename, MIME subtype, file size, and upload date. Page size scales with viewport width.

### Adaptive Page Sizes

The number of assets loaded per page automatically increases as the viewport gets wider so large and 4K monitors are used efficiently:

| Viewport width | Large tiles | Small tiles | List |
|----------------|-------------|-------------|------|
| < 1920 px | 24 | 48 | 25 |
| 1920 – 2559 px (Full HD / QHD) | 48 | 96 | 50 |
| ≥ 2560 px (4K+) | 96 | 192 | 100 |

Page size is re-evaluated automatically when the browser window is resized (debounced 300 ms). A new API request is made only when the size tier actually changes — not on every pixel of resize.

<Callout kind="info">
  The Asset Library uses the full available width of the content area with no maximum-width cap, so every column of pixels is used for asset previews on wide monitors.
</Callout>

### File-Type Icons

Non-image assets display **color-coded Lucide icons** based on their MIME type instead of a generic document icon:

| File Type | Icon | Color |
|-----------|------|-------|
| Audio (MP3, WAV, etc.) | `file-audio` | Violet |
| Video (MP4, WebM, etc.) | `file-video` | Rose |
| PDF | `file-text` | Red |
| Archives (ZIP, RAR, etc.) | `file-archive` | Yellow |
| Code (JSON, HTML, etc.) | `file-code` | Emerald |
| Spreadsheets (Excel, CSV) | `file-spreadsheet` | Green |
| Presentations (PowerPoint) | `presentation` | Orange |
| Documents (Word, TXT) | `file-text` | Blue |
| Fonts | `file-type` | Amber |
| SVG | `image` | Pink |

### Media Previews

The detail drawer provides native playback for media files:

- **Video assets** — HTML5 `<video>` player with browser-native controls
- **Audio assets** — file-audio icon with HTML5 `<audio>` player controls beneath

### API Search

```bash
GET /api/v1/assets?search=hero&mimeType=image/&sortBy=createdAt&sortOrder=desc&page=1&limit=48
```

<Callout kind="tip">
  The `limit` parameter on the assets endpoint accepts values up to **250** (higher than the standard API cap of 100) to support large-screen tile grids.
</Callout>

### Facets

Get counts of assets by type:

```bash
GET /api/v1/assets/facets
# Returns: { images: 234, videos: 12, audio: 5, documents: 45, other: 3 }
```

## Asset Metadata

Each asset has editable metadata:

| Field | Description |
|-------|-------------|
| **Title** | Display name for the asset |
| **Description** | Longer description text |
| **Alt Text** | Accessibility text for images |
| **Custom Metadata** | Free-form JSON metadata |

```bash
PUT /api/v1/assets/:id/metadata
{
  "title": "Climate Summit 2026",
  "description": "Opening ceremony photo",
  "altText": "Delegates at the climate summit opening ceremony",
  "custom": { "photographer": "Jane Doe", "license": "CC BY 4.0" }
}
```

## Usage Tracking

Studio tracks which content items reference each asset. This helps you:

- See where an asset is used before deleting it
- Find unused assets that can be cleaned up
- Understand asset impact across your content

```bash
# Check which content references this asset
GET /api/v1/assets/:id/usage

# Check if an asset can be safely deleted
GET /api/v1/assets/:id/can-delete

# Find unused assets
GET /api/v1/assets/unused?olderThan=2026-01-01&limit=50
```

## Storage Limits

| Edition | Storage Limit | Per-File Limit |
|---------|--------------|----------------|
| **CE** | 5 GB total | 50 MB per file |
| **Pro** | Unlimited | Configurable |

Check current storage usage:

```bash
GET /api/v1/assets/storage
# Returns: { usedBytes: 1073741824, limitBytes: 5368709120, percentUsed: 20.0 }
```

## Storage Providers

Studio uses an **S3-compatible storage** interface. Configure your storage provider in `.env`:

```bash
STORAGE_PROVIDER=local    # Options: local, s3
S3_ENDPOINT=https://s3.eu-central-1.amazonaws.com
S3_BUCKET=studio-assets
S3_ACCESS_KEY=your-access-key
S3_SECRET_KEY=your-secret-key
S3_REGION=eu-central-1
```

<Callout kind="tip">
  For development, the `local` storage provider stores files on disk. For production, use S3-compatible storage (AWS S3, Hetzner Object Storage, MinIO, etc.).
</Callout>

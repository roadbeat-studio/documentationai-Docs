## Overview

roadbeat Studio is configured primarily through **environment variables** set in a `.env` file or via your deployment platform. All configuration is validated on startup using Joi schemas — missing required variables cause the application to fail fast with a clear error message.

## Required Variables

These variables must be set for Studio to start:

| Variable | Description | Example |
|----------|-------------|---------|
| `DATABASE_URL` | PostgreSQL connection string | `postgresql://user:pass@localhost:5432/studio` |
| `JWT_SECRET` | Secret key for signing JWT tokens (min 32 chars) | `your-secret-key-at-least-32-characters` |

## Database

| Variable | Default | Description |
|----------|---------|-------------|
| `DATABASE_URL` | — | PostgreSQL connection string |

Studio uses **Prisma** as its ORM. The database URL follows the format:

```
postgresql://USER:PASSWORD@HOST:PORT/DATABASE?schema=public
```

<Callout kind="tip">
  For production, enable connection pooling with PgBouncer or Prisma's built-in connection pool. Add `?connection_limit=10` to the DATABASE_URL.
</Callout>

## Redis

| Variable | Default | Description |
|----------|---------|-------------|
| `REDIS_HOST` | `localhost` | Redis server hostname |
| `REDIS_PORT` | `6379` | Redis server port |
| `REDIS_PASSWORD` | — | Redis password (optional) |
| `REDIS_DB` | `0` | Redis database number |

Redis is used for caching, session storage, and BullMQ job queues.

## Authentication

| Variable | Default | Description |
|----------|---------|-------------|
| `JWT_SECRET_CURRENT` | — | **Primary** secret for signing new JWT tokens |
| `JWT_SECRET_PREVIOUS` | — | **Previous** secret — tokens signed with this key are still accepted (for zero-downtime rotation) |
| `JWT_SECRET` | — | Legacy single-secret fallback (used if `JWT_SECRET_CURRENT` is not set) |
| `JWT_EXPIRATION` | `24h` | Access token lifetime |
| `JWT_REFRESH_EXPIRATION` | `7d` | Refresh token lifetime |

<Callout kind="tip">
  **Key rotation procedure:** To rotate JWT secrets without logging out all users:
  1. Copy the current `JWT_SECRET_CURRENT` value to `JWT_SECRET_PREVIOUS`
  2. Set `JWT_SECRET_CURRENT` to a new random string
  3. Restart the API — existing tokens are verified against both secrets
  4. After the access token lifetime (default 24h), all active tokens will use the new secret
  5. Remove `JWT_SECRET_PREVIOUS` on the next rotation cycle
</Callout>

## Mail

| Variable | Default | Description |
|----------|---------|-------------|
| `MAIL_TRANSPORT` | `log` | Mail transport: `smtp` or `log` (dev-only, prints to console) |
| `MAIL_FROM` | `noreply@localhost` | Sender address for outbound emails |
| `MAIL_BASE_URL` | `http://localhost:4200` | Base URL for links in email templates (e.g. password reset) |
| `SMTP_HOST` | — | SMTP server hostname (required when `MAIL_TRANSPORT=smtp`) |
| `SMTP_PORT` | `587` | SMTP server port |
| `SMTP_USER` | — | SMTP username |
| `SMTP_PASS` | — | SMTP password |

<Callout kind="info">
  When `MAIL_TRANSPORT=log` (the default), emails are printed to the server console instead of being sent. This is useful for local development — you can see password reset links in the logs without configuring an SMTP server.
</Callout>

<Callout kind="tip">
  For production, set `MAIL_TRANSPORT=smtp` and configure your SMTP credentials. Services like Postmark, SendGrid, Mailgun, or Amazon SES all work with standard SMTP settings.
</Callout>

## Search

| Variable | Default | Description |
|----------|---------|-------------|
| `MEILISEARCH_HOST` | `http://localhost:7700` | MeiliSearch server URL |
| `MEILISEARCH_API_KEY` | — | MeiliSearch master key |

<Callout kind="info">
  MeiliSearch is optional. Without it, Studio falls back to PostgreSQL full-text search.
</Callout>

## CDN (Optional)

| Variable | Default | Description |
|----------|---------|-------------|
| `CDN_ENABLED` | `false` | Enable CDN URL rewriting |
| `CDN_URL` | — | CDN base URL |
| `CDN_PROVIDER` | `generic` | CDN provider: `generic`, `cloudfront`, `cloudflare`, `bunny` |
| `CDN_SIGNING_KEY` | — | HMAC key for signed URLs |
| `CDN_DEFAULT_TTL` | `86400` | Default cache TTL in seconds |

## Federation (Optional)

| Variable | Default | Description |
|----------|---------|-------------|
| `FEDERATION_GATEWAY_URL` | — | Base URL of the Federation Gateway service (e.g. `https://federation.roadbeat.net`). When not set, the Federation tab in Platform Admin shows a configuration hint and all federation features are disabled. |
| `FED_ADMIN_TOKEN` | — | Shared secret for authenticated communication between Studio and the Federation Gateway. Must match the `ADMIN_TOKEN` configured on the gateway. Use a long random string (min 64 chars). |

<Callout kind="info">
  Federation is entirely optional. Studio works normally without these variables — the Federation tab simply shows that the gateway is not configured. See the [Federation guide](/guides/federation) for setup instructions.
</Callout>

| Variable | Default | Description |
|----------|---------|-------------|
| `LOG_LEVEL` | `debug` | Pino log level: `trace`, `debug`, `info`, `warn`, `error`, `fatal` |

## Multi-Tenancy

| Variable | Default | Description |
|----------|---------|-------------|
| `CANONICAL_URL_BASE` | — | Fallback base URL for published content canonical URLs (per-org override in `organization.settings.canonicalUrlBase` takes priority) |
| `UPLOAD_DIR` | `./uploads` | Root directory for asset uploads (org files stored under `{orgSlug}/` subdirectories) |

<Callout kind="tip">
  Most multi-tenancy settings are configured per-organization via the Platform Admin dashboard rather than environment variables. See [Multi-Tenancy](/guides/multi-tenancy) for details.
</Callout>

## Rate Limiting

| Variable | Default | Description |
|----------|---------|-------------|
| `THROTTLE_TTL` | `60000` | Rate limit window in ms |
| `THROTTLE_LIMIT` | `100` | Max requests per window per org+IP |

<Callout kind="info">
  Rate limiting is **org-aware**. The throttle key is `org:{orgId}:{ip}` for authenticated requests, so one organization's traffic cannot exhaust another's rate limit budget.
</Callout>

## Example .env File

```bash
# Database
DATABASE_URL=postgresql://studio:secretpassword@localhost:5432/roadbeat_studio

# Redis
REDIS_HOST=localhost
REDIS_PORT=6379

# Auth
JWT_SECRET_CURRENT=change-this-to-a-long-random-string-at-least-64-chars
# JWT_SECRET_PREVIOUS=old-secret-during-key-rotation
JWT_EXPIRATION=24h

# Search (optional)
MEILISEARCH_HOST=http://localhost:7700
MEILISEARCH_API_KEY=your-meili-master-key

# Storage
STORAGE_PROVIDER=local

# Publishing
SCHEMA_REGISTRY_URL=http://localhost:3200
DISCOVERY_NODE_URLS=http://localhost:3300

# Consumer
CONTEXT_DIRECTORY_URL=http://localhost:3100

# Federation (optional — enables ActivityPub)
# FEDERATION_GATEWAY_URL=http://localhost:3500
# FED_ADMIN_TOKEN=change-this-to-a-long-random-token-at-least-64-chars

# Mail
MAIL_TRANSPORT=log
MAIL_FROM=noreply@localhost
MAIL_BASE_URL=http://localhost:4200
# SMTP_HOST=smtp.example.com
# SMTP_PORT=587
# SMTP_USER=your-smtp-user
# SMTP_PASS=your-smtp-password

# Server
PORT=3000
CORS_ORIGIN=http://localhost:4200
NODE_ENV=development
LOG_LEVEL=debug
LOG_PRETTY=true
```

## Production Checklist

<Callout kind="alert">
  Before deploying to production, ensure these items are addressed:
</Callout>

- [ ] `JWT_SECRET_CURRENT` is a unique, random string (min 64 chars)
- [ ] Enable MFA for all admin accounts (TOTP via authenticator app)
- [ ] `DATABASE_URL` uses a strong password
- [ ] `NODE_ENV=production` is set
- [ ] `LOG_PRETTY=false` for structured JSON logs
- [ ] `CORS_ORIGIN` is set to your actual domain
- [ ] `STORAGE_PROVIDER=s3` with proper credentials
- [ ] HTTPS is configured (via Caddy, nginx, or load balancer)
- [ ] Database backups are scheduled
- [ ] Redis persistence is enabled (`appendonly yes`)
- [ ] MeiliSearch master key is set
- [ ] `MAIL_TRANSPORT=smtp` with valid SMTP credentials
- [ ] `MAIL_FROM` uses your domain (e.g. `noreply@yourdomain.com`)
- [ ] `MAIL_BASE_URL` matches your public Studio URL
- [ ] `FEDERATION_GATEWAY_URL` points to a secured gateway (HTTPS, not publicly reachable without auth)
- [ ] `FED_ADMIN_TOKEN` is a unique, random string (min 64 chars)

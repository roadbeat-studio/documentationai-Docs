## Welcome, Editor

This guide is for **content editors** — the people who create, edit, and publish content in roadbeat Studio. You don't need to be a developer to use Studio effectively. This section covers everything you need to know to be productive.

## Your Workspace

When you log in to Studio, you'll see the **Dashboard** with:

- **Quick stats** — total published items, drafts, and content breakdown by type
- **Recent content** — your most recently edited items for quick access
- **Status overview** — a breakdown of content by status (draft, published, archived)

### Sidebar Navigation

The sidebar on the left is your main navigation:

| Section | What it does |
|---------|-------------|
| **Dashboard** | Overview stats and recent activity |
| **Content** | Browse, create, and edit content items |
| **Content Types** | View available content types (admin-managed) |
| **Assets** | Media library — images, videos, documents |
| **Users** | User management (admin only) |
| **Settings** | Organization settings (admin only) |

<Callout kind="tip">
  The sidebar also shows a **DISCOVER** section with consumer features like Compass, Discover, Publishers, and Bookmarks — these let you explore content from the broader roadbeat network.
</Callout>

## Daily Workflow

A typical editor's daily workflow looks like this:

<Steps>
  <Step title="Check your dashboard" icon="layout-dashboard">
    Review recent content and any items that need attention.
  </Step>
  <Step title="Create or edit content" icon="edit">
    Open an existing item or create new content. The editor generates the right form for each content type.
  </Step>
  <Step title="Upload media" icon="image">
    Add images, videos, or documents to the Asset Library, then reference them in your content.
  </Step>
  <Step title="Review and publish" icon="check-circle">
    Review your content, then click **Publish** to make it live.
  </Step>
</Steps>

## Quick Actions

| Shortcut | Action |
|----------|--------|
| **Ctrl+K** / **Cmd+K** | Open global search (command palette) |
| **Ctrl+S** / **Cmd+S** | Save current content |
| Click content type in sidebar | Jump to content list filtered by that type |

## Key Concepts for Editors

### Content Types

Each piece of content belongs to a **content type** (e.g., News, Event, Recipe). The content type determines which fields appear in the editor form. You don't need to worry about creating content types — that's done by admins.

### Drafts and Publishing

- Every new item starts as a **Draft**
- Drafts are only visible to logged-in Studio users
- Click **Publish** to make content live and available via the API
- Click **Unpublish** to take content offline

### Versions

Every time you save, a new **version** is created. You can:

- View the version history
- Compare changes between versions
- Restore a previous version if needed

### Auto-Save

Studio auto-saves your work every 5 seconds while you're editing. You'll see a "Saved" indicator in the status bar. If you navigate away with unsaved changes, a warning dialog appears.

<Update label="2026-05-21" description="v0.11.0" tags={["feature"]}>
  ## ActivityPub Federation

  Organizations can now **federate via ActivityPub**. When enabled, each organization gets a Fediverse handle that users on Mastodon, Lemmy, Pleroma, and other compatible servers can follow. Published content is automatically delivered to remote followers.

  **Publisher lifecycle:**
  - Provision a publisher actor from the Platform Admin ΓåÆ Organization Detail ΓåÆ Federation tab
  - Enable/disable outbound federation with a single toggle
  - Fediverse handle, IRI, and delivery counters displayed on the Identity sub-tab

  **Inbound interactions (6 sub-panels in the Federation tab):**
  - **Followers** ΓÇö paginated follower list with accept/reject/remove actions and status filtering
  - **Discussion** ΓÇö inbound replies with status tabs (pending, approved, rejected, dropped) and per-reply approve/reject moderation
  - **Reactions** ΓÇö like and boost analytics with totals, top-objects table, and paginated activity stream (analytics-only, never on consumer cards)
  - **Moderation** ΓÇö block and flag event queue with kind ├ù status filters, resolve/dismiss with notes, and colour-coded cards
  - **Policy** ΓÇö configurable follow policy (auto-accept / manual approve) and reply policy (auto-approve / manual approve / drop) with radio card UI

  **Security:**
  - HTTP Signatures (Cavage draft-12) for all outbound/inbound verification
  - AES-256-GCM encrypted key storage with optional Vault Transit backend
  - Redis-backed replay cache for duplicate activity rejection
  - Server-side HTML sanitisation of inbound reply content

  **API endpoints:**
  - Publisher lifecycle: `GET/POST/PUT /platform/organizations/:orgId/federation`
  - Followers: `GET /ΓÇª/followers`, `POST /ΓÇª/followers/:id/accept`, `POST /ΓÇª/followers/:id/reject`
  - Replies: `GET /ΓÇª/replies`, `POST /ΓÇª/replies/:id/approve`, `POST /ΓÇª/replies/:id/reject`
  - Reactions: `GET /ΓÇª/reactions/analytics`, `GET /ΓÇª/reactions`
  - Moderation: `GET /ΓÇª/moderation`, `POST /ΓÇª/moderation/:id/resolve`, `POST /ΓÇª/moderation/:id/dismiss`
  - Policy: `PATCH /ΓÇª/policy`
</Update>

  ## Publisher Muting

  Users can now **mute publishers** to permanently hide their content from all Discover views ΓÇö including goal-driven results, search, "all", and "random" modes.

  **New capabilities:**
  - Mute/unmute publishers from the publisher card kebab menu (Γï«), publisher detail page, or "Why am I seeing this?" transparency drawer
  - Muted publishers tab in the publisher directory with count badge
  - Muted badge indicator on publisher cards
  - Mutual exclusivity: muting a followed publisher automatically unfollows them (and vice versa)

  **Technical:**
  - `localStorage` persistence (`rb_muted_publishers`) with 5-second debounced server sync
  - Backend: `muted_publisher_ids` column on `cd_account_links` table with full CRUD endpoints at `/consumer/follows/muted`
  - Discovery Node exclusion via `publisher.id_exclude` filter on all federated searches
  - Bookmarks from muted publishers remain accessible
</Update>

<Update label="2026-05-02" description="v0.10.8" tags={["feature", "bugfix"]}>
  ## Content Editor ΓÇö Video Picker, Country/Currency Selects, Location Improvements

  **Video Picker component** (`ui.component: "video_picker"`)

  - New dual-mode video field: select a video from the **Asset Library** or paste an external URL (YouTube, Vimeo, or direct video)

  - Footer badge indicates the source: **From asset library** (green), **YouTube embed** / **Vimeo embed** / **External URL** (blue)

  - Added `video` to the `FieldType` union for form schema field definitions

  **Country Select** (`ui.component: "country_select"`)

  - New dropdown populated from Context Directory via `GET /api/v1/countries` proxy endpoint

  **Currency Select** (`ui.component: "currency_select"`)

  - New dropdown populated from Context Directory via `GET /api/v1/currencies` proxy endpoint

  - `GET /api/v1/countries` ΓÇö proxies Context Directory `/api/v1/countries` with 1-hour cache and built-in fallback list (EU + US countries)

  **Location Search improvements**

  - Search component now always appears **above** the address fields to match the natural address-entry flow
  - Country Select dropdown is auto-filled when a location is selected from search results
  - `LocationSelectedEvent` extended with `streetName`, `streetNumber`, and `countryName` fields

  **Dynamic enum resolution**

  - Editor field renderer now resolves enum options from the content type's metadata schema when an `i18n_key` is specified on the field ΓÇö eliminates the need for hardcoded option lists in the frontend

  ## Content Pod ΓÇö Auto-Publish & Dashboard Content List

  **Automatic pod publish from the Content Editor**

  - Publishing content from the editor now automatically publishes to the Content Pod if an active pod is configured for the organization ΓÇö no separate step required
  - Two toast notifications confirm the operation: "Content published" followed by "Published to Content Pod"
  - A pod publish failure never blocks the Studio publish ΓÇö the content remains available via the Delivery API and can be re-pushed manually

  **Pod Dashboard content tab fixed**

  - The Content tab now reads directly from `manifest.json` on the pod filesystem via the new `GET /api/v1/manage/content-pod/items` endpoint
  - Previously relied on the Pod Hosting proxy (external service at port 3400), which caused "No content published to the pod yet" even when content existed in the pod
  - Each row now shows: title, locale badge, content type, slug, bundle size, publish date, and a direct "View Γåù" link to the pod URL
  - The Refresh button re-fetches the manifest; the trash icon unpublishes the item from the pod

  **New API endpoint**

  - `GET /api/v1/manage/content-pod/items` ΓÇö returns all entries from the pod manifest (`id`, `contentType`, `slug`, `locale`, `title`, `sizeBytes`, `publishedAt`, `podUrl`, etc.)

  ## Multi-Tenancy ΓÇö Zero Data Leakage

  roadbeat Studio is now fully multi-tenant. Multiple organizations share a single database and application instance with strict data isolation.

  **Database Foundation (Phase 1)**

  - New `platform_admins` table for super admin tracking
  - `content_types` table refactored to composite primary key `(organization_id, id)` ΓÇö orgs can have content types with the same IDs
  - All foreign keys from `content` and `content_type_layouts` updated to composite references
  - Removed `organization_content_types` junction table (no longer needed)

  **Authentication & Platform Module (Phase 2)**

  - Multi-org email resolution ΓÇö login handles users with the same email across multiple organizations (org picker)
  - `setupInitialAdmin()` creates the first platform admin and default locale
  - Platform admin API: organization CRUD, platform stats, super admin grant/revoke
  - `superAdminGuard` for platform-level route protection

  **Core Services (Phase 3)**

  - `ContentTypesService` ΓÇö all methods require `organizationId`; composite PK queries (`organizationId_id`)
  - `SearchService` ΓÇö org-scoped `clearIndexForOrg()`, `getIndexStatsForOrg()`, `reindexForOrg()`
  - `SearchController` ΓÇö `reindex()` and `stats()` scoped to requesting user's org
  - `DiscoveryNodesService`, `PublishingService`, `PublisherKeyService`, `DeliveryService`, `DashboardService` ΓÇö all org-scoped
  - `ContentRelationsService` ΓÇö cross-org validation hardened on back-references, usage, and delete checks
  - Cache keys audited ΓÇö all 8 org-scoped calls include `${organizationId}` in key
  - Asset storage keys prefixed with `{orgSlug}/` for human-readable paths
  - `HealthController` ΓÇö removed `process.uptime()` from public endpoints; queue stats behind auth
  - `canonicalUrlBase` reads from `organization.settings` first, falls back to env var
  - `ExportModule` (`BackupService`, `ContentPodService`) verified org-scoped
  - `FeatureGateService.getLimit()` reads per-org overrides via `OrgLimitsLoaderService`
  - `OrgThrottlerGuard` ΓÇö rate limit key = `org:{orgId}:{ip}` for per-org throttling
  - Asset serving guard middleware ΓÇö checks JWT org against path org dir; super-admin bypass

  **Pro Plugin Isolation (Phase 4)**

  - `pro-realtime` ΓÇö WebSocket rooms org-prefixed; `sendToContentRoom()` requires `organizationId`; REST admin endpoints return per-org data; all `@OnEvent` handlers guard missing `organizationId`
  - `pro-scheduling` ΓÇö `organization_id` added to `pro_scheduled_actions`; all queries org-scoped
  - `pro-import` ΓÇö `organization_id` added to `pro_imported_content`

  **Frontend (Phases 5ΓÇô6)**

  - Sidebar org header with logo and name
  - Super-admin org-switcher dropdown
  - Settings org tab with plan, limits, and domain configuration
  - Content Types enable/disable tab for per-org content type management
  - Platform Admin dashboard: organization list with search/filter/pagination, org detail with users/stats/settings tabs, create-org wizard with 4-step flow
  - Platform stats component with KPI cards, orgs-by-plan chart, recent orgs
  - Platform admins component with admin table, grant/revoke with user search modal
</Update>

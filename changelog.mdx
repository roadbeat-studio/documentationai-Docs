<Update label="2026-03-16" description="v0.9.8" tags={["feature"]}>
## 3D Globe view in Discover

**Mapbox 3D Globe**
- **Interactive globe** — the Discover page now includes a 3D globe view mode powered by Mapbox GL JS with the `globe` projection, using the colorful `streets-v12` style with atmosphere, stars, and fog effects
- **Custom emoji markers** — each content type is represented by a unique circular marker with a white background, colored border, and emoji icon (🎉 Event, 📰 News, 🎪 Festival, 💼 Job, 🍳 Recipe, 🏠 Property, 🎓 Course, 🛍️ Product, 📍 default). Markers scale up smoothly on hover
- **Compact popup cards** — clicking a marker opens a 260px-wide popup card with: hero image (120px, rounded corners) with close button overlay, outlined content type badge, publisher name, bold title, truncated description, and a footer with publication date and info/bookmark action buttons
- **Smart camera positioning** — the globe automatically flies to the geographic center of all markers, with zoom level calculated from the coordinate span (single marker → zoom 6, continent-wide → zoom 1.5)
- **Optional spin** — globe auto-rotation is disabled by default but can be toggled via the spin button; rotation pauses during user interaction and slows at higher zoom levels

**Backend: Mapbox configuration**
- **Public mapbox-config endpoint** — new `GET /consumer/discover/mapbox-config` controller serves the Mapbox token without authentication, gated by the `MAPBOX_TOKEN` environment variable
- **Globe availability** — when no Mapbox token is configured, the globe view mode is automatically hidden from the view toggle bar
</Update>

<Update label="2026-03-15" description="v0.9.7" tags={["feature"]}>
## Interactive map view in Discover

**Map view**
- **Leaflet + OpenStreetMap** — the Discover page now includes a fully interactive map view using Leaflet with OpenStreetMap tiles, accessible via the map icon in the view mode toggle bar
- **Marker clustering** — nearby markers are grouped into cluster badges (blue &lt;10, amber 10–99, red 100+) that expand on zoom or click, powered by Leaflet.markercluster
- **Vertical popup cards** — clicking a marker opens a rich popup card matching the teaser grid card design: full-width hero image, content type badge, publisher name, title (2-line clamp), description (2-line clamp), publication date, and info/bookmark action buttons
- **Facet sidebar visible** — unlike the calendar view, the map renders inside the content area alongside the facet sidebar, so users can filter by content type, language, topic, publisher, and date while viewing the map
- **Auto-fit bounds** — the map automatically zooms to fit all visible markers with padding, up to zoom level 14
- **Reactive to filters** — changing any filter, goal tab, or search query dynamically updates the markers on the map
- **Dark mode support** — popup cards, cluster badges, and action buttons adapt to dark mode

**Backend geo data pipeline**
- **TeaserDto extended** — `location` field now includes `lat`, `lon`, and `name` for geographic coordinates
- **mapHitToTeaser extraction** — the BFF extracts `geographic.locations[0].coordinates` from Discovery Node responses and maps them to the teaser DTO
- **extractGeographic fix** — the teaser generator now correctly handles both structured `geographic` objects and single `location` objects with `latitude`/`longitude` coordinate keys
- **Coordinate format fix** — standardized on `lon` (matching Elasticsearch `geo_point`) instead of `lng` throughout the backend
</Update>

<Update label="2026-03-14" description="v0.9.6" tags={["feature", "bugfix"]}>
## Goal-driven search & bookmarking

**Goal-based content filtering**
- **All Goals active by default** — the Discover page now uses the user's entire goal portfolio to filter content on initial load, combining all content_queries into a single batched Elasticsearch `bool.should` query for sub-100ms performance
- **Six search modes** — All Goals (default), Goal Group (category tab), Single Goal, Browse All (unfiltered), Random/Surprise, and Followed Publishers Only (toggle filter)
- **Batched ES query architecture** — instead of one HTTP request per content_query (which would mean 150+ requests for 50 goals), all queries are sent in a single request to the Discovery Node's new `POST /search/by-goals` endpoint
- **Random/Surprise mode** — "✨ Random" button uses Elasticsearch `function_score` with `random_score` for serendipitous content discovery, with optional light bias from user's goal categories
- **"All" button** — browse all content without goal filtering, using manual facet filters and full-text search
- **Content_queries resolved at portfolio load** — goal definitions are fetched from the Context Directory once (cached in Redis for 30 minutes), `{{param}}` placeholders substituted with user values, and attached to each goal for instant use in searches

**Bookmark popover**
- **Collection selector** — clicking the bookmark icon on any card opens a centered modal dialog showing all bookmark collections with checkboxes
- **Last-used collection pre-selected** — the most recently used collection is automatically checked for new bookmarks (persisted in localStorage)
- **Inline collection creation** — "New collection" button reveals an inline text input; created collections immediately appear in the list and get selected
- **Update & remove** — already-bookmarked teasers show "Update" and "Remove bookmark" options
- **Teaser preview** — the dialog shows the content title, type, and publisher for confirmation

**Discover UX improvements**
- **Scrollable facet lists** — filter groups with more than 8 options are now height-limited (240px) with overflow scrolling and a search-as-you-type input that filters by label or key
- **Publication date filter** — new dropdown with preset timespans (Today, Last 7 days, Last 30 days, This month, This quarter, This year) and custom from/to date pickers with locale-aware formatting
- **DN request debug panel** — collapsible `<details>` block below search results showing the exact JSON payloads sent to each Discovery Node, for Elasticsearch query debugging during development
- **Goal tab filter fix** — resolved "property filters should not exist" validation error when clicking goal tabs by adding `filters` and `contentQueries` to `GoalSearchDto`
- **Filter path mapping** — goal catalog paths (`category.tags`, `category.categories`) correctly mapped to DN classification paths (`classification.tags`, `classification.categories`); content-type-specific fields like `event_type` are ignored since the teaser generator doesn't populate them
</Update>

## Compass & Discover UX improvements

- **Drag-and-drop goal reordering** — replaced up/down arrow buttons on goal cards with Angular CDK drag-and-drop, allowing users to reorder goals by dragging within their Compass portfolio
- **Segment-based editing in goal editor** — when editing an existing goal, the standardized goal text with inline variable inputs is now shown (same as the add flow), with the full title editable below; parameter changes automatically update the resolved goal name
- **Two-level goal group tabs in Discover** — the Discover page now shows goal category tabs (e.g., "Business & Organisation Goals", "Personal Goals") as a first level, with individual goals as a second-level tab row beneath; tab order follows the user's Compass sort order
- **Improved Discovery Node error messages** — the BFF now includes the Axios error code (e.g., `ECONNREFUSED`) when a Discovery Node is unreachable, instead of showing an empty error message
- **8 view modes in Discover** — expanded view mode switching from 3 to 8 options: Activity Feed, Compact List, Large List, Compact Tiles, Large Tiles, Calendar, Map, and 3D Globe
- **Activity Feed view** — new social-media-style feed as the first view mode, with timeline connector, publisher avatar, big hero images, relative timestamps, and engagement action bar (Why this, Save, Share)
- **Large List view** — new detailed list rows with 192×128px thumbnails, full 3-line descriptions, content type badges, publisher name, location with region, and relevance score
- **Large Tiles view** — new 3-per-row card grid with taller 16:10 images, overlay action buttons, badge chips with overflow count, and a footer with location and relevance score
- **Map & 3D Globe placeholders** — toggle buttons for Map and 3D Globe views with styled "Coming soon" placeholder panels (detailed implementations to follow)
- **Grouped view mode toggle** — the view mode switcher is now split into 5 visually grouped button clusters: Feed | Lists | Tiles | Calendar | Geo, with spacing between groups for better scannability
</Update>

<Update label="2026-03-13" description="v0.9.3" tags={["feature"]}>
## Transparency drawer improvements

- **Goal matching in drawer** — the "Why am I seeing this?" drawer now displays which of the user's goals relate to each content item, using a three-tier matching strategy: direct goal-based search match, active goal tab, and fuzzy matching against badges and content type
- **Dynamic goal match indicator** — green "Goal match" box when goals are matched, blue "Content match" box with contextual guidance when no goals match or none are set up
- **Matched goals display** — green badges with checkmark icons showing goal name and category, or informative empty-state messages ("No specific goal matched" / "No goals set up yet")
- **Navigation links** — "Edit your goals" now navigates to the Compass page, "Adjust location settings" navigates to Profile Settings
- **Goal ID tagging** — teasers from goal-based searches now carry the originating `goalId` for guaranteed match attribution
</Update>

<Update label="2026-03-13" description="v0.9.4" tags={["feature", "bugfix"]}>
## Unified Discover Settings & API fixes

- **Unified Discover Settings** — merged Account Settings and Consumer Settings into a single page at `/consumer/settings` with 7 card sections: Discover Profile, Language & Region, Privacy, Notifications, Content Preferences, Discovery Nodes, and Security
- **Dual-account UX concept** — info banner explaining the relationship between the Studio account (CMS admin) and the Discover profile (roadbeat network identity), with a link to System → Settings for Studio-level configuration
- **CD profile API fix** — corrected Context Directory profile endpoint from `/api/v1/users/me` (404) to `/api/v1/auth/profile`, with field mapping (`username` → `email`, `displayName` → `name`)
- **Resilient profile loading** — `GET /api/v1/consumer/user` now returns Studio user data as primary source, enriching with CD data when available instead of failing if the CD is unreachable
- **Notifications migration** — added `notifications` and `notification_preferences` tables to the Studio database with indexes, foreign keys, and auto-created default preferences
- **Settings in sidebar** — added "Settings" link with sliders icon to the Discover section of the Studio sidebar navigation
- **Profile page consolidation** — simplified profile page settings links from 6 entries to 3 (Discover Settings, Sync & App, My Data) with correct `/consumer/*` paths
- **SVG icon fix** — replaced broken link icon on the "Linked to roadbeat network" indicator with a proper check-circle icon
</Update>

<Update label="2026-03-11" description="v0.9.2" tags={["feature"]}>
## Compass horizon filtering

- **Horizon tabs in category detail** — time horizon filter tabs (All, Life, Year, Month, Week, Day) now appear when viewing a single category's goals, with the goal count updating to reflect filtered results
- **Grouped goal list on overview** — selecting a specific horizon on the main Compass page shows a flat goal list grouped by category name instead of category cards, with only categories containing matching goals displayed
- **Horizon normalization fix** — abbreviated time horizons (`l`, `y`, `m`, `w`, `d`) are now correctly matched against full-word filters (`life`, `year`, `month`, `week`, `day`) across all views
</Update>

<Update label="2026-03-10" description="v0.9.1" tags={["feature"]}>
## Goal editor redesign

- **Multi-select goals** — Step 2 now supports selecting multiple goals at once with checkbox toggles and a selection count badge
- **Per-goal customization** — Step 3 displays a scrollable list of selected goal cards, each with inline parameter inputs and integrated time horizon selection
- **Goal removal** — Step 4 (Confirm) shows goal cards with an X button to remove individual goals before saving
- **Bulk save API** — New `POST /api/v1/consumer/goals/bulk` endpoint for creating multiple goals in a single request
- **Inline parameter inputs** — Goal text templates render with embedded text fields and searchable dropdowns for parameters with suggestions
- **Time horizon preselection** — Automatically selects horizon when a goal has exactly one `time_context` value; shows suggested horizons with grey background when multiple options exist
- **Horizon normalization** — Short forms (`y`, `m`, `w`, `d`, `l`) are normalized to full words (`year`, `month`, `week`, `day`, `life`) throughout the UI
- **Compact goal cards** — Goal list cards now show parameters and time horizon on a single line with the horizon badge right-aligned
- **Search results mapping** — Fixed MeiliSearch camelCase-to-snake_case field mapping for goal search results
</Update>

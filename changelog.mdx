<Update label="2026-05-20" description="v0.11.0" tags={["feature"]}>
  ## Content Pod Auto-Provisioning & Quota Enforcement

  **Free Pod on Org Creation**

  - New organizations created via `POST /platform/organizations` can include `podMode: 'static' | 'dynamic'` to auto-provision a free Content Pod
  - `ContentPodSetupService` handles provisioning: creates `ContentPodConfig`, sets subdomain to org slug, configures storage paths, deploys dynamic scripts for dynamic pods
  - Response includes `podSetup` with mode, tier, subdomain, baseUrl, and status

  **ContentPod Module**

  - New `content-pod/` module: `ContentPodConfigService` (CRUD), `ContentPodPublishingService` (publish/unpublish with quota enforcement), `ContentPodSetupService` (free pod provisioning), `ContentPodController` (REST endpoints)
  - Prisma: `ContentPodConfig` model with `podMode`, `tier`, `dynamicRuntime`, `dynamicConfig` (JSON), `quotaStorageBytes` (BigInt), `quotaItemCount` (Int), `publishedCount`, `totalSizeBytes`, `lastPublishedAt`
  - Endpoints: `GET/PUT /manage/content-pod/config`, `POST /manage/content-pod/publish/:id`, `POST /manage/content-pod/unpublish/:id`, `GET /manage/content-pod/test`

  **Quota Enforcement**

  - Publishing returns HTTP 403 when item count or storage quota exceeded
  - Tier-based defaults: free (500 MB / 200), starter (5 GB / 5K), pro (20 GB / 50K), enterprise (unlimited)
  - Admin override via `PUT /manage/content-pod/config` with custom `quotaItemCount` and `quotaStorageBytes`

  **Onboarding Status**

  - `GET /platform/onboarding/status` now includes `contentPod` section with pod mode, tier, quota limits, and current usage

  **Platform Integration**

  - `PlatformModule` imports `ContentPodModule`
  - `CreateOrganizationDto` extended with optional `podMode`
  - `PlatformService` and `OnboardingService` call setup service after org creation
</Update>

<Update label="2026-04-22" description="v0.10.4" tags={["bugfix"]}>
  ## Slug Field Deduplication

  **Removed duplicate slug from SEO tab**

  - The SEO tab previously rendered a plain text input for the slug field (`identifier.slug`) via the common SEO section schema ŌĆö this field was disconnected from the actual slug state managed by the content editor, meaning edits to it were silently discarded on save
  - The `identifier.slug` field has been removed from the SEO section; the SEO tab now contains only meta title, meta description, keywords, canonical URL, OG image, and no-index toggle

  **Explicit slug field for no-title content types**

  - Content types without a primary title field (`quote`, `check-in`, `joke`, `social-media-post`, `dating-profile`, `glossary-entry`, `private-user-profile`) now have an explicit `identifier.slug` field in their first form section
  - The content editor intercepts `identifier.slug` fields in all three layout modes (tabs, sections, flat) and renders them via the `rb-editor-slug-inline` component ŌĆö the same component used for slug auto-generation below title fields ŌĆö ensuring the slug is properly connected to the editor's slug state and saved correctly
</Update>

  - Removed `organization_content_types` junction table (no longer needed)

  **Authentication & Platform Module (Phase 2)**

  - Multi-org email resolution ŌĆö login handles users with the same email across multiple organizations (org picker)
  - `setupInitialAdmin()` creates the first platform admin and default locale
  - Platform admin API: organization CRUD, platform stats, super admin grant/revoke
  - `superAdminGuard` for platform-level route protection

  **Core Services (Phase 3)**

  - `ContentTypesService` ŌĆö all methods require `organizationId`; composite PK queries (`organizationId_id`)
  - `SearchService` ŌĆö org-scoped `clearIndexForOrg()`, `getIndexStatsForOrg()`, `reindexForOrg()`
  - `SearchController` ŌĆö `reindex()` and `stats()` scoped to requesting user's org

  - `ContentRelationsService` ŌĆö cross-org validation hardened on back-references, usage, and delete checks
  - Cache keys audited ŌĆö all 8 org-scoped calls include `${organizationId}` in key
  - Asset storage keys prefixed with `{orgSlug}/` for human-readable paths
  - `HealthController` ŌĆö removed `process.uptime()` from public endpoints; queue stats behind auth
  - `canonicalUrlBase` reads from `organization.settings` first, falls back to env var
  - `ExportModule` (`BackupService`, `ContentPodService`) verified org-scoped
  - `FeatureGateService.getLimit()` reads per-org overrides via `OrgLimitsLoaderService`
  - `OrgThrottlerGuard` ŌĆö rate limit key = `org:{orgId}:{ip}` for per-org throttling
  - Asset serving guard middleware ŌĆö checks JWT org against path org dir; super-admin bypass

  **Pro Plugin Isolation (Phase 4)**

  - `pro-realtime` ŌĆö WebSocket rooms org-prefixed; `sendToContentRoom()` requires `organizationId`; REST admin endpoints return per-org data; all `@OnEvent` handlers guard missing `organizationId`
